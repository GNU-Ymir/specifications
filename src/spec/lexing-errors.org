* Ymir Lexical Specification — Error Codes
** Overview
Lexing errors represent violations of the lexical grammar or malformed source
constructs detected before parsing. Each lexing error is assigned a *stable,
machine‑readable error code* used by diagnostics, IDE tooling, and automated
analysis.

This document defines:
- the error‑code format,
- the classification categories,
- the mapping between error codes and error messages.

** Error Code Format
Lexing error codes follow the format:

#+begin_example
LEX.<CATEGORY>.<NUMBER>
#+end_example

Where:
- =LEX= identifies the lexing subsystem,
- =CATEGORY= identifies the error family,
- =NUMBER= is a zero‑padded integer unique within the category.

Example:
#+begin_example
LEX.STRING.001
#+end_example

Error codes are stable across compiler versions.

** Categories
The following categories are defined for lexing:

| Category  | Meaning                                      |
|-----------+----------------------------------------------|
| STRING    | String literal errors                        |
| COMMENT   | Comment block and documentation errors       |
| ESCAPE    | Escape‑sequence errors                       |
| NUMBER    | Numeric literal errors                       |
| IDENT     | Identifier and name errors                   |
| VERSION   | Version‑directive (#if/#else/#end) errors    |
| GENERAL   | Miscellaneous lexing errors                  |

The following table defines the mapping between error codes and the
=LexingErrorMessage= enumeration.

|------------------+------------------------------------+--------------------------------------------------------|

| LEX.STRING.002   | RETURN_IN_SINGLE_LINE_STRING       | "line break encountered inside a single-line string"  |
| LEX.STRING.003   | ESCAPE_RETURN_PROHIBITED_STRING    | "escaping a line break is prohibited; use a multiline string instead @(y=\"\"\")" |
| LEX.STRING.004   | ESCAPE_AT_END_OF_STRING            | "escape character found at end of string literal"     |
| LEX.STRING.005   | EMPTY_DEC_FLOAT_NUMBER             | "the fractional part of the floating-point number is empty" |

|------------------+------------------------------------+--------------------------------------------------------|
| LEX.VERSION.001  | UNTERMINATED_VERSION_BLOCK         | "unterminated version block: missing @(y=#end)"       |

*** GENERAL Errors
(Currently unused; reserved for future lexing diagnostics.)

** Error Reporting Requirements
A conforming Ymir implementation must:
1. Emit the error code and message together.
2. Include the source location of the offending word.
3. Include notes for secondary locations when provided.
4. Preserve error‑code stability across compiler versions.

Example diagnostic:

#+begin_example
Error[LEX.STRING.001]: unterminated string literal "
   --> file.yr:12:5
    |
 12 |     let s = "hello
    |             ^ missing closing quote
#+end_example

** Versioning
New lexing errors introduced in future versions must:
- extend the appropriate category,
- use the next available integer,
- update this specification and the versioned change log.
